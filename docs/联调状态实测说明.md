## 待联调的接口(按联调文档全流程)

前端需按 `docs/联调文档.md` 接入以下流程:

1. **单图上传** `POST /exams/upload-image` → 拿到真实图片 URL
2. **提交试卷** `POST /exams/upload-by-urls`(传真实 URL) → 立即返回 exam_id
3. **轮询识别** `GET /exams/{id}/recognition` → 等 `recognized`
4. **确认** `POST /exams/{id}/confirm` → 立即返回 `analyzing`
5. **轮询分析** `GET /exams/{id}/analysis` → 等 `analyzed`
6. **出题** `POST /practice/generate` → 立即返回 `generating`
7. **轮询题目** `GET /practice/{id}/questions` → 等 `ready`
8. **提交答案** `POST /practice/{id}/submit` → 立即返回 `assessing`
9. **轮询评估** `GET /practice/{id}/assessment` → 等 `assessed`

## 关键验证点

用**真实图片**走完整流程(步骤 1-9),确认:
- 每个提交接口都是**立即返回**(不超时)
- 每个轮询接口最终能拿到**完成态**(`recognized`/`analyzed`/`ready`/`assessed`)
- 如果某个轮询一直停在中间态,记录是哪个接口、哪个状态,反馈给后端
