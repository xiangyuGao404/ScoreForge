# ScoreForge 前端修复任务清单

**目标**：前端从 Mock 模式切换到真实后端联调模式，修复所有阻塞对接的问题。
**后端地址**：`http://localhost:8000`（已启动）
**后端 API 文档**：`http://localhost:8000/docs`

---

## P0 — 必须先做（阻塞联调）

### 任务 1：创建 `.env` 文件，默认关闭 Mock

**文件**：`frontend/.env`

```
VITE_USE_MOCK=false
```

> 当前 `api.ts:8` 通过 `import.meta.env.VITE_USE_MOCK !== 'false'` 控制。不设置时默认 `true`（Mock 模式）。创建 `.env` 后重启 `npm run dev:h5` 即可连真实后端。

---

### 任务 2：添加 Vite 开发代理

**文件**：`frontend/vite.config.ts`

当前文件内容只有：
```ts
import { defineConfig } from "vite";
import uni from "@dcloudio/vite-plugin-uni";
export default defineConfig({ plugins: [uni()] });
```

需要添加 `server.proxy`，把 `/api/v1` 转发到后端：

```ts
import { defineConfig } from "vite";
import uni from "@dcloudio/vite-plugin-uni";

export default defineConfig({
  plugins: [uni()],
  server: {
    proxy: {
      "/api/v1": {
        target: "http://localhost:8000",
        changeOrigin: true,
      },
    },
  },
});
```

### 任务 3：修复 `uploadExam` 的真实上传逻辑

**文件**：`frontend/src/utils/service.ts` 第 55-75 行

**当前问题**：即使 `USE_MOCK=false`，图片上传仍然走 `mock.delay(200)`，传给后端的是本地文件路径而非真实图片。

**需要改为**：使用 `uni.uploadFile`（api.ts 中已有封装）逐张上传图片到后端，拿到服务端返回的图片 URL，再组装请求体调用 `/exams/upload`。

**参考**：`api.ts` 中的 `uploadFile` 函数已经封装好了 multipart 上传逻辑，直接调用即可。

### 任务 9：添加全局错误处理

**文件**：`src/main.ts`

**当前问题**：无 `app.config.errorHandler`，异步错误静默丢失。

**需要添加**：
```ts
app.config.errorHandler = (err, instance, info) => {
  console.error('[Global Error]', err, info)
  uni.showToast({ title: '发生错误，请重试', icon: 'none' })
}
```

---

### 任务 10：添加路由守卫

**文件**：`src/main.ts` 或 `src/App.vue`

**当前问题**：未登录可直接访问所有页面。

**需要实现**：在 `App.vue` 的 `onLaunch` 中检查 token，无 token 时 `uni.redirectTo('/pages/login/index')`。已在 `App.vue:9` 有部分逻辑，但跳转缺失。

### 任务 11：修复 401 处理不清理 store

**文件**：`src/utils/api.ts` 第 42-44 行

**当前问题**：401 时只清 localStorage 的 token，未调用 `userStore.logout()`，导致 `isLoggedIn` 仍为 `true`。

**需要改为**：在 401 handler 中导入并调用 `useUserStore().logout()`。

## 与后端对接的关键信息

| 项目 | 说明 |
|------|------|
| 登录验证码 | DEBUG 模式下固定 `888888` |
| Token 格式 | JWT，放在 `Authorization: Bearer {token}` |
| 响应格式 | `{code: 0, message: "...", data: {...}}` |
| 错误格式 | `{code: 400, message: "...", detail: "..."}` |
| 学生列表 | `GET /api/v1/students` → `{data: {students: [...]}}` |
| 薄弱点列表 | `GET /api/v1/weaknesses?student_id=xxx` |
| 试卷上传 | `POST /api/v1/exams/upload`（multipart/form-data） |
