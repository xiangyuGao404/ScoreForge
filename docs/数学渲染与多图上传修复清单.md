## 问题 1：数学符号渲染

### 现象

AI 返回的题目中包含 LaTeX 格式（如 `$x \geq 6$`、`$x^2 + 1$`），在前端直接显示为原始文本，不渲染。

### 方案决策

**V1.0 使用：Prompt 约束 + 前端 Unicode 转换**（兼容小程序 + H5）
**V2.0 可选：引入 KaTeX**（仅 H5，+85KB gzipped，小程序不支持 DOM 渲染）

### 修复方案（V1.0：Prompt 约束 + 前端兜底转换）

#### 后端：修改所有出题 Prompt，禁止使用 LaTeX

**文件**：`backend/app/services/ai_service.py`

在 `generate_questions` 的 System Prompt 中添加：

```
## 数学公式格式要求
- 禁止使用 LaTeX 格式（如 $x^2$、\frac{a}{b}）
- 使用纯文本 + Unicode 数学符号表示：
  - 平方：x²（上标 Unicode）
  - 下标：x₁
  - 根号：√x
  - 大于等于：≥
  - 小于等于：≤
  - 不等于：≠
  - 分数：用文字描述 "二分之一" 或 "1/2"
  - 乘号：×（不用 *）
  - 除号：÷（不用 /）
  - 圆周率：π
  - 角度：∠ABC
- 如果必须表示复杂公式，用文字描述而非符号
```

#### 前端：添加 LaTeX → Unicode 转换函数（兜底）

**文件**：新建 `frontend/src/utils/math.ts`

```ts
/**
 * 将简单 LaTeX 数学表达式转换为 Unicode 文本
 * 处理 AI 输出中残留的 $...$ 格式
 */

const LATEX_MAP: Record<string, string> = {
  '\\geq': '≥', '\\leq': '≤', '\\neq': '≠',
  '\\times': '×', '\\div': '÷', '\\pm': '±',
  '\\sqrt': '√', '\\pi': 'π', '\\infty': '∞',
  '\\alpha': 'α', '\\beta': 'β', '\\gamma': 'γ',
  '\\angle': '∠', '\\degree': '°',
  '\\approx': '≈', '\\equiv': '≡',
  '\\rightarrow': '→', '\\leftarrow': '←',
  '\\Rightarrow': '⇒',
  '\\quad': ' ', '\\qquad': '  ',
  '\\text{': '', '\\mathrm{': '',
  '\\left': '', '\\right': '',
  '\\(': '', '\\)': '', '\\[': '', '\\]': '',
}

// 上标映射
const SUPERSCRIPT: Record<string, string> = {
  '0': '⁰', '1': '¹', '2': '²', '3': '³', '4': '⁴',
  '5': '⁵', '6': '⁶', '7': '⁷', '8': '⁸', '9': '⁹',
  'n': 'ⁿ', 'i': 'ⁱ', '+': '⁺', '-': '⁻', '(': '⁽', ')': '⁾',
}

// 下标映射
const SUBSCRIPT: Record<string, string> = {
  '0': '₀', '1': '₁', '2': '₂', '3': '₃', '4': '₄',
  '5': '₅', '6': '₆', '7': '₇', '8': '₈', '9': '₉',
  'a': 'ₐ', 'e': 'ₑ', 'i': 'ᵢ', 'j': 'ⱼ',
  'n': 'ₙ', 'x': 'ₓ',
}

function convertSupSub(match: string, type: '^' | '_', content: string): string {
  const map = type === '^' ? SUPERSCRIPT : SUBSCRIPT
  return content.split('').map(c => map[c] || c).join('')
}

export function latexToText(text: string): string {
  if (!text) return text

  let result = text

  // 替换 \frac{a}{b} → a/b
  result = result.replace(/\\frac\{([^}]*)\}\{([^}]*)\}/g, '$1/$2')

  // 替换 \sqrt{x} → √(x)
  result = result.replace(/\\sqrt\{([^}]*)\}/g, '√($1)')

  // 替换 x^{n} → xⁿ
  result = result.replace(/\^{([^}]*)}/g, (_, content) =>
    content.split('').map(c => SUPERSCRIPT[c] || c).join('')
  )
  // 替换 x^n → xⁿ（单字符）
  result = result.replace(/\^([0-9n])/g, (_, c) => SUPERSCRIPT[c] || c)

  // 替换 x_{n} → xₙ
  result = result.replace(/_{([^}]*)}/g, (_, content) =>
    content.split('').map(c => SUBSCRIPT[c] || c).join('')
  )
  // 替换 x_n → xₙ（单字符）
  result = result.replace(/_([0-9nx])/g, (_, c) => SUBSCRIPT[c] || c)

  // 替换 LaTeX 命令
  for (const [cmd, replacement] of Object.entries(LATEX_MAP)) {
    result = result.replace(new RegExp(cmd.replace(/\\/g, '\\\\'), 'g'), replacement)
  }

  // 移除残留的花括号
  result = result.replace(/\{([^{}]*)\}/g, '$1')

  // 移除 $ 包裹
  result = result.replace(/\$\$(.*?)\$\$/g, '$1')
  result = result.replace(/\$(.*?)\$/g, '$1')

  // 清理多余空格
  result = result.replace(/\s{2,}/g, ' ').trim()

  return result
}
```

## 问题 2：多图上传只识别第一张

### 现象

上传 3 张试卷图片，只有第一张被识别。

### 根因

**前端**：`service.ts` 第 122 行 `filePath: params.images[0]` 只传了第一张图。
**后端**：`ocr_service.py` 的 `recognize_exam` 接收的是 `image_urls` 列表，理论上支持多张，但前端只传了一张。

### 修复

#### 前端：循环上传所有图片

**文件**：`frontend/src/utils/service.ts` 的 `uploadExam` 函数

**当前代码**（第 119-127 行）：
```ts
return new Promise<any>((resolve, reject) => {
    uni.uploadFile({
      url: '/api/v1/exams/upload',
      filePath: params.images[0], // ← 只传第一张
      name: 'images',
      formData: { ... },
```

**改为**：
```ts
// 逐张上传图片，收集服务端返回的 URL
async function uploadImages(images: string[], token: string): Promise<string[]> {
  const urls: string[] = []
  for (const img of images) {
    const res = await new Promise<any>((resolve, reject) => {
      uni.uploadFile({
        url: '/api/v1/exams/upload-image',  // 新端点：单张图片上传
        filePath: img,
        name: 'image',
        header: { Authorization: `Bearer ${token}` },
        success: (r) => {
          if (r.statusCode === 200) {
            resolve(JSON.parse(r.data))
          } else {
            reject(new Error('图片上传失败'))
          }
        },
        fail: (e) => reject(new Error(e.errMsg || '上传失败')),
      })
    })
    urls.push(res.data.url)
  }
  return urls
}
```

**或者更简单的方案**：改后端接收方式，让前端一次传完所有图片。

#### 后端：新增单图上传端点 + 修改试卷上传接收方式

**方案 A（推荐）**：前端逐张上传到新端点，拿到 URL 列表后调用 `/exams/upload`

```
POST /api/v1/exams/upload-image
Content-Type: multipart/form-data
Body: image (单个文件)
Response: { "code": 0, "data": { "url": "/uploads/xxx.jpg" } }
```

然后前端：
```ts
// 1. 先逐张上传图片，拿到 URL 列表

// 2. 再调用试卷上传接口，传 URL 列表

    subject: params.subject,
    exam_name: params.exam_name,
    total_score: params.total_score,
    actual_score: params.actual_score,
    image_urls: imageUrls,  // URL 列表

```

**方案 B**：后端改为接收多个文件

当前后端已经是 `images: list[UploadFile] = File(...)`，理论上支持多文件。问题是前端只传了一个。需要前端用 `uni.uploadFile` 的循环方式传。
