## 第一部分：阻塞本地启动的问题（必须先修）

> 以下问题不修，项目在本地跑不起来。

### 🔴 B-1：`redis[hiredis]` 在 Windows 上安装失败

**文件**：`backend/requirements.txt:12`
**现象**：`pip install -r requirements.txt` 报错，hiredis C 扩展编译失败
**修复**：改为 `redis==5.0.4`（去掉 `[hiredis]`，纯 Python 驱动够用）

### 🔴 B-2：`paddlepaddle==2.6.1` 不支持 Python 3.13

**文件**：`backend/requirements.txt:27-28`
**现象**：pip install 找不到匹配的 wheel，编译也失败
**修复**：从 requirements.txt 移除 `paddleocr` 和 `paddlepaddle`，本地用 mock 模式（代码已自动 fallback）

### 🔴 B-3：`weasyprint==62.3` 依赖系统级 C 库（Pango/Cairo）

**文件**：`backend/requirements.txt:33`
**现象**：Windows 上 pip install 可能成功，但运行时 import 报 `OSError`
**修复**：从 requirements.txt 移除 `weasyprint`，PDF 功能暂不可用；或安装 GTK for Windows

### 🔴 B-4：没有 PostgreSQL，所有端点 500

**文件**：`backend/app/core/database.py:8` + `backend/app/api/deps.py:38`
**现象**：uvicorn 能启动，`/` 和 `/health` 正常，但任何需要登录的端点都报 `ConnectionRefusedError`
**修复**：
- 方案 A：本地安装 PostgreSQL，创建数据库，运行 `alembic upgrade head`
- 方案 B：改用 SQLite（需改 `database.py` 和 `requirements.txt` 加 `aiosqlite`）

### 🔴 B-5：没有 Redis，登录流程 500

**文件**：`backend/app/api/v1/auth.py:41,51,80`
**现象**：`send-code` 和 `login` 端点无条件调用 Redis，无 Redis 时 `ConnectionError`。注意：`888888` 开发码虽然存在，但 Redis 调用在它之前执行，所以开发码实际上无法生效
**修复**：
- 方案 A：本地安装 Redis
- 方案 B：auth 端点的 Redis 调用改为可选（`if redis available else skip`）

### 🔴 B-6：没有 `.env` 文件，数据库密码等使用默认值

**文件**：`backend/.env.example` → 需复制为 `backend/.env`
**现象**：应用静默使用占位符密钥，连接默认数据库地址
**修复**：`cp .env.example .env`，按本地环境修改

### 🔴 F-1：`static/` 目录缺失，Tab 栏图标不存在

**文件**：`frontend/src/pages.json:80-101`
**现象**：`npm run dev:h5` 构建时 uni-app Vite 插件读取 pages.json 找不到图标文件，可能报错或 Tab 栏空白
**修复**：创建 `frontend/src/static/tab/` 目录，放入 8 个 PNG 图标（任意占位图即可）

### 🔴 F-2：`@vue/tsconfig` 版本过低，TypeScript 配置可能失败

**文件**：`frontend/package.json:29` + `frontend/tsconfig.json:1`
**现象**：`@vue/tsconfig@^0.1.3` 不包含 `tsconfig.json` 导出路径，`extends` 解析失败
**修复**：升级到 `"@vue/tsconfig": "^0.3.0"`，同时 TypeScript 升级到 `"^5.0.0"`

## 第二部分：运行时会崩溃的问题

> 项目能启动，但点某些功能会报错。

### 🟠 R-1：WeasyPrint 生成 PDF 时 500

**文件**：`backend/app/services/pdf_service.py:183`
**触发**：调用 `POST /api/v1/pdf/generate`
**现象**：`from weasyprint import HTML` 在 Windows 上因缺少 Pango/Cairo 报 `OSError`
**影响**：PDF 功能完全不可用（如果 B-3 选择移除 weasyprint 则此问题不存在）

### 🟠 R-2：AI 服务无 API Key，所有 AI 端点返回固定 Mock 数据

**文件**：`backend/app/services/ai_service.py:24-36,377-434`
**触发**：调用 `/exams/{id}/analysis`、`/practice/generate`、`/practice/{id}/assessment`
**现象**：`settings.OPENAI_API_KEY` 为 `None`，`_get_client()` 返回 `None`，走 mock 分支
**影响**：功能"能用"但每次返回完全相同的数据，无法验证真实逻辑
**说明**：这是设计如此（开发模式），不算 bug，但需要知道

### 🟠 R-3：数据库表不存在，首次启动所有查询失败

**文件**：`backend/app/main.py:22-30`
**触发**：PostgreSQL 连上了但没建表
**现象**：`relation "users" does not exist`
**修复**：启动后运行 `alembic upgrade head`（注意 `alembic.ini:6` 硬编码了数据库 URL，需要与 `.env` 一致）

### 🟠 R-4：前端 `practice/index.vue` 赋值语句有 copy-paste 错误

**文件**：`frontend/src/pages/practice/index.vue:139`
**现象**：`answers.value[currentIndex.value = currentIndex.value] = opt` 自赋值，功能不受影响但代码有误
**影响**：不会崩溃，但表明代码未经仔细审查

### 🟠 R-5：前端 `practice/result.vue` "标记已掌握"按钮未调用 API

**文件**：`frontend/src/pages/practice/result.vue:107-114`
**触发**：AI 评估建议"已掌握"后，用户点击按钮
**现象**：只 `showToast` + `navigateTo('/pages/home/index')`，未调用 `masterWeakness()`
**影响**：用户以为标记了，但服务端状态未更新

### 🟠 R-6：前端 `chat/index.vue` 模块顶层调用异步函数

**文件**：`frontend/src/pages/chat/index.vue:171`
**现象**：`switchRole('math')` 在 `<script setup>` 顶层执行，组件未挂载就发请求
**影响**：Mock 模式下无问题；切真实后端时如果请求失败，unhandled promise rejection

## 本地启动清单（按顺序执行）

```
步骤 1：修后端依赖
  - requirements.txt: redis[hiredis] → redis
  - requirements.txt: 移除 paddleocr, paddlepaddle
  - requirements.txt: 移除 weasyprint（或装 GTK）
  - requirements.txt: bcrypt 4.1.3 → bcrypt>=4.2.0

步骤 2：准备后端环境
  - 安装 PostgreSQL 16，创建数据库 scoreforge 和用户
  - 安装 Redis（或用 Docker）
  - cp .env.example .env，修改 DATABASE_URL 和 REDIS_URL
  - pip install -r requirements.txt
  - alembic upgrade head（建表）

步骤 3：启动后端
  - cd backend
  - uvicorn app.main:app --reload --port 8000
  - 访问 http://localhost:8000/health 确认启动

步骤 4：修前端
  - 创建 frontend/src/static/tab/ 放 8 个占位图标
  - package.json: @vue/tsconfig 升级到 ^0.3.0
  - package.json: typescript 升级到 ^5.0.0
  - npm install

步骤 5：启动前端
  - cd frontend
  - npm run dev:h5
  - 浏览器打开显示的地址
  - 当前为 Mock 模式，所有功能可用（数据是假的）
```
