#### 试卷上传流程（核心流程）

```
步骤 1： 循环调用 POST /exams/upload-image
         上传每张图片 → 拿到 {url}
         （每张约 1-3 秒，图片会自动压缩到 500KB 以内）

步骤 2： 调用 POST /exams/upload-by-urls
         Body: {student_id, subject, exam_name, total_score, actual_score, image_urls: [url1, url2, url3]}
         → 立即返回 {"status": "recognizing", "exam_id": "xxx"}
         （不等 OCR 完成，约 200ms 返回）

步骤 3： 跳到确认页，轮询 GET /exams/{id}/recognition
         每 2.5 秒查一次
         返回 {"status": "recognized", "questions": [...]} 时停止轮询
         （OCR 约 15-60 秒完成）

步骤 4： 家长审查题目对错，点击确认
         调用 POST /exams/{id}/confirm
         Body: {questions: [{question_no, is_correct, score_got, parent_verified}]}
         → 立即返回 {"status": "analyzing"}

步骤 5： 跳到分析页，轮询 GET /exams/{id}/analysis
         每 2.5 秒查一次
         返回 {"status": "analyzed", "weaknesses": [...]} 时停止轮询
         （AI 分析约 30-90 秒完成）
```

## 三、轮询工具函数

```typescript
/**
 * 轮询直到条件满足或超时
 * @param fn   每轮执行的异步函数
 * @param check 判断是否完成的函数，返回 true 停止轮询
 * @param interval 轮询间隔，默认 2500ms
 * @param maxAttempts 最大轮询次数，默认 60（约 2.5 分钟）
 */
export function poll<T>(
  fn: () => Promise<T>,
  check: (data: T) => boolean,
  interval = 2500,
  maxAttempts = 60
): Promise<T> {
  return new Promise((resolve, reject) => {
    let attempts = 0
    const timer = setInterval(async () => {
      attempts++
      try {
        const data = await fn()
        if (check(data)) {
          clearInterval(timer)
          resolve(data)
        } else if (attempts >= maxAttempts) {
          clearInterval(timer)
          reject(new Error('处理超时，请稍后重试'))
        }
      } catch (e) {
        clearInterval(timer)
        reject(e)
      }
    }, interval)
  })
}
```

### 轮询使用示例

```typescript
// 轮询识别结果
const res = await poll(
  () => getRecognition(examId),
  (data) => data.data.status === 'recognized',
  2500, 60 // 每 2.5 秒，最多 60 次
)

// 轮询分析结果
const res = await poll(
  () => getAnalysis(examId),
  (data) => data.data.status === 'analyzed',
  2500, 60
)

// 轮询题目生成
const res = await poll(
  () => getPracticeQuestions(sessionId),
  (data) => data.data.status === 'ready',
  2500, 60
)

// 轮询评估结果
const res = await poll(
  () => getAssessment(sessionId),
  (data) => data.data.status === 'assessed',
  2500, 60
)
```

## 五、注意事项

### 5.1 Token 传递

所有需要登录的接口都要在 Header 中传 Bearer Token：
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

Token 从 `POST /auth/login` 的响应中获取，有效期 7 天。

### 5.2 开发模式

- 验证码固定为 `888888`（任何手机号均可登录）
- 前端 `.env` 需设置 `VITE_USE_MOCK=false` 才会调用真实后端
- CORS 已开放 `localhost:5173`、`localhost:3000`、`localhost:8080`

### 5.3 超时设置

- 异步接口（upload-by-urls、confirm、generate、submit）：30 秒超时即可
- 轮询接口（recognition、analysis、questions、assessment）：无需超时，由轮询控制
- 教师对话（chat/send）：建议 60 秒超时

### 5.4 图片上传

- 图片在后端会自动压缩（缩放到 1600px，压缩到 500KB 以内）
- 支持格式：`jpg`、`jpeg`、`png`、`bmp`、`webp`
- 单张最大 10MB，每次最多 5 张

### 5.5 数学符号

- AI 出题使用 Unicode 符号（如 `x²`、`≥`、`√`、`π`），不使用 LaTeX
- 前端 `src/utils/math.ts` 中有 `latexToText()` 函数做兜底转换
- 所有显示题目内容的地方都应调用 `latexToText()`
