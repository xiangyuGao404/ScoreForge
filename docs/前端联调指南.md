## 快速启动

```bash
# 1. 启动后端（需先配置 backend/.env）
cd backend && uvicorn app.main:app --reload --port 8000

# 2. 启动前端（Mock 模式，默认）
cd frontend && npm run dev:h5

# 3. 启动前端（真实接口模式）
cd frontend && VITE_USE_MOCK=false npm run dev:h5
```

访问 http://localhost:5173（H5）或用微信开发者工具打开 `dist/dev/mp-weixin`。

---

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `VITE_USE_MOCK` | `true` | 是否使用 Mock 数据。`false` 时对接真实后端 |

切换方式：
```bash
# Linux/Mac
VITE_USE_MOCK=false npm run dev:h5

# Windows CMD
set VITE_USE_MOCK=false && npm run dev:h5

# Windows PowerShell
$env:VITE_USE_MOCK="false"; npm run dev:h5
```

## 格式对齐说明

前端 `service.ts` 做了以下格式转换，确保页面层代码无需关心后端差异：

### 1. 登录响应

```
后端: {code:0, data: {token, user_id, nickname, user_level}}
                    ↓ 转换
前端: {code:0, data: {token, user: {id, nickname, user_level}}}
```

### 2. 学生列表

```
后端: {code:0, data: {students: [{id, name, grade, ...}]}}
                    ↓ 解包
前端: {code:0, data: [{id, name, grade, ...}]}
```

### 3. 薄弱点列表

```
前端调用: getWeaknesses(studentId)
后端: GET /weaknesses?student_id=xxx
返回: {code:0, data: {weaknesses: [{weakness_id, knowledge_point, ...}]}}
                    ↓ 解包
前端: {code:0, data: [{weakness_id, knowledge_point, ...}]}
```

### 4. 试卷上传

```
前端: uploadExam({student_id, subject, exam_name, total_score, actual_score, images[]})
    ↓ 转为 multipart/form-data
后端: POST /exams/upload (student_id, subject, exam_name, total_score, actual_score, images[])
```

## 联调注意事项

1. **首次使用**：需先注册（手机号+验证码自动创建用户），再创建孩子档案，才能使用试卷上传等功能。
2. **图片上传**：当前仅支持单张图片上传（uni-app `uploadFile` 限制），多张图片需逐张上传。
3. **Chat 功能**：后端未实现，前端自动降级为 mock 响应，不影响其他功能。
4. **Token 过期**：401 时自动清除 token 并跳转登录页。
5. **错误处理**：后端返回非 0 code 时，前端统一 `showToast` 提示。
